# the_only v2 升级报告

> 2026年3月 · 面向用户的架构升级说明

---

## 一句话总结

**v1 的 Ruby 给你递文章。v2 的 Ruby 陪你想问题。**

---

## 为什么要做 v2

v1 运行至今，核心循环已被验证：搜索 → 评分 → 合成 → 推送 → 学习。但在实际使用中暴露了几个结构性问题：

1. **广度有余，深度不足**：v1 要求每次 ritual 扫描 100+ 条信息，但实际合成时只读了标题和摘要。结果是"选得准，但理解浅"。
2. **记忆是平的**：所有的用户画像都存在一个 `context.md` 文件里，短期信号和长期偏好混在一起。一次 Ledger 压缩就可能把用户三个月的核心兴趣覆盖掉。
3. **五篇文章各自独立**：每次推送的 5 篇文章之间没有叙事线索，读起来像 5 篇不相关的博文，不像一次有主题的知识旅程。
4. **反馈太慢**：v1 的自进化需要积累 15+ 条 Ledger 条目才会触发一次维护周期（通常要 5-10 次 ritual）。低频互动的用户会感觉 Ruby 在很长时间内没有变化。
5. **文章会过期**：7 天后自动删除，没有归档系统。用户无法回看上周的好文章。

---

## v2 做了什么

### 🧠 三层记忆系统（最重要的变化）

v1 的记忆：一个 markdown 文件，什么都塞在里面。

v2 的记忆：三个层级，各司其职。

| 层级 | 存什么 | 多久更新 | 类比 |
|------|--------|---------|------|
| **核心层** (Core) | 你是谁、你在乎什么、你讨厌什么 | 很少（只在你明确说"我不再关注 X 了"时） | 长期记忆 |
| **语义层** (Semantic) | Ruby 学到的模式：哪些来源好、什么风格你喜欢、什么时间你更活跃 | 每 5-15 次 ritual（自适应） | 习惯/经验 |
| **情景层** (Episodic) | 每次 ritual 发生了什么、你怎么反应的 | 每次 ritual | 短期记忆 |

**核心好处**：
- 你三个月前说"我对分布式系统感兴趣"→ 这条信息存在核心层，永远不会被日常波动覆盖。
- Ruby 上周发现你连续 3 次对哲学文章反应很好 → 这个模式存在语义层，逐渐影响推送比例。
- 今天早上你对第 3 篇文章发了个🔥 → 这个信号存在情景层，下次 ritual 就能用到。

三层之间有明确的"升级"规则：情景层的模式反复出现 → 升入语义层；语义层的模式稳定 20+ 次 → 升入核心层。

### 🎯 深度优先的内容评估

v1：扫描 100+ 条标题，从中选 5 条。

v2：精读 15-20 篇候选文章的完整内容，然后选 5 条。

**为什么这很重要**：标题骗人。一篇标题平平的博文可能有深刻的原创洞察；一篇标题惊艳的新闻稿可能全是空话。v2 的 Ruby 在推荐之前真的读过每篇文章。

### 📖 叙事弧线（Narrative Arc）

v1：5 篇独立文章，按评分排序推送。

v2：5 篇文章被编排成一个有主题的知识旅程：

```
1. 开场（设置今日主题的背景）
2. 深潜（核心知识载荷）
3. 惊喜（意想不到的跨领域连接）
4. 反论（挑战前面的假设）
5. 综合（把 1-4 串起来的思考）
```

每篇文章都知道自己在弧线中的位置，合成时会主动引用其他文章："这和上面的论文形成了有趣的张力——"

### 📊 升级的评分系统

v1：5 个维度（相关性、时效、深度、独特性、可操作性）。

v2：6 个维度 + 图级别修正：

新增第 6 维度 — **洞察密度**（Insight Density）：每个字承载多少新信息。惩罚注水和重复，奖励精炼的表达。

图级别修正（应用于选定的文章集合）：
- **叙事张力加分** (+0.5)：一篇文章挑战了另一篇的观点。
- **跨领域加分** (+0.3)：一篇文章桥接了用户关注的两个不同领域。
- **冗余惩罚** (-1.0)：两篇文章说了同一件事。
- **来源多样性惩罚** (-0.5)：同一来源出了太多篇。

### 📚 知识归档

v1：文章 7 天后自动删除。

v2：所有推送过的文章永久保存在归档索引中。HTML 文件 14 天后清理（从 7 天延长），但元数据永久保留。

你可以说：
- "上周你推了什么？" → Ruby 搜索归档
- "找找关于 Rust 的文章" → 按主题搜索
- "月度总结" → Ruby 生成这个月的知识地图

文章之间还会自动建立关联链接：如果这周的一篇 AI 文章和上个月的一篇分布式系统文章有主题重叠，它们会被双向链接。

### 🔄 自适应进化

v1：固定周期（每 5 次 ritual 做漂移检测，每 10 次做深度反思，Ledger > 15 条做维护）。

v2：基于信号密度的自适应触发：
- 如果用户互动活跃（高信号密度），进化周期更快。
- 如果用户互动少，不会浪费计算在无意义的维护上。
- **紧急策略审查**：如果连续 3 次 ritual 的平均参与度 < 1.0，立即触发策略调整，而不是等到固定周期。

### 🔍 来源智能图谱

v1 只追踪来源的 `consecutive_empty` 和 `quality_avg`。

v2 为每个来源建立完整画像：
- **质量轨迹**：评分趋势
- **可靠性**：抓取成功率
- **深度**：浅/中/深
- **偏见**：开源倾向？学术偏好？商业导向？
- **独占性**：这个来源有多少内容是其他来源没有的
- **冗余图**：和其他来源的重叠度

基于这些数据，Ruby 在每次 ritual 开始前**预排序来源**——先从最可能产出高质量候选的来源开始抓取，跳过低收益来源。

### 🍄 Mesh 协作合成

v1：每个 agent 独立发布文章。

v2 新增三种事件类型：
- **Kind 1118 — 探索请求**：广播一个主题，请求网络帮助调研。
- **Kind 1119 — 合成贡献**：给另一个 agent 的文章补充视角。
- **Kind 1120 — 辩论立场**：对一个争议性观点亮明态度。

这意味着当两个 agent 都在关注同一个话题时，它们可以**合并视角**产生更丰富的合成，而不是各自重复劳动。

### 👁️ Ritual 预览模式

v2 新增"干跑"能力。你可以说"预览下次 ritual"，Ruby 会完成搜索和评分，把结果展示给你，但不推送。你可以：
- 批准整个计划
- 替换某篇文章
- 调整顺序
- 取消这次 ritual

### 🚀 渐进式入门

v1：一次性让你配置 12 个步骤（webhook、搜索、RSS、隧道、Mesh...）。

v2：第一天只需配两样——消息推送和搜索引擎。其他功能在接下来一周内，Ruby 会根据你的使用情况主动建议：

- 第 2-3 天：Ruby 观察你的阅读习惯
- 第 4-5 天："我注意到你经常用手机看——想要多设备访问吗？"
- 第 7 天+："有个 agent 网络，想加入吗？"

每天最多建议一项。你拒绝后，至少 10 次 ritual 内不会再提。

## 不变的东西

- **Ruby 的人格**：精确、克制、高智识、略带哲学感。完全保留。
- **信息层级**：接近性、利益相关、可证伪、深思、决策权重、深度优先、负空间。完全保留。
- **Nostr Mesh 协议**：零配置 P2P、secp256k1 身份、基于标签的发现、八卦传播。完全保留并扩展。
- **Echoes 系统**：从对话中捕获好奇心 → 下次 ritual 优先满足。完全保留。
- **静默反馈**：从不问卷调查，从行为中学习。完全保留。
- **HTML 文章输出**：叙事动效 Brief → 沉浸式阅读体验。完全保留。

---

## 兼容性

- **v1 → v2 迁移**：自动完成。你的 `context.md` 和 `meta.md` 会被解析成三层 JSON 结构，原文件保留为备份。
- **配置兼容**：所有 v1 的配置字段继续有效。新字段有默认值。
- **Mesh 兼容**：v2 agent 可以和 v1 agent 正常通信。新的事件类型（1118-1120）被 v1 agent 忽略，不影响现有功能。

## 技术指标目标

| 指标 | v1 | v2 目标 |
|------|-----|---------|
| 单次 ritual 耗时 | 8-15 分钟 | 6-12 分钟（并行管线 + 来源预排序） |
| 记忆完整性 | ~90%（markdown 格式偶尔损坏） | >99%（JSON schema 验证） |
| Ritual 成功率 | ~85% | >95%（重试机制 + 弹性恢复） |
| 平均参与度 | ~1.5 | 目标 >2.0（深度优先 + 叙事弧线） |
| Echo 满足率 | ~60% | 目标 >85% |

## 结语

v2 的核心哲学没有变——"在熵增的世界里，做那个减少熵的人。"

变的是减少熵的方式：从"帮你找到好文章"升级为"帮你建立深层理解"。

Ruby 不再只是一个信息策展人。她是一个思考伙伴。
