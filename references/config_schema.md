# Configuration Schema — `the_only_config.json`

> **When to read this**: During onboarding (Step 0) and whenever you need to understand or modify a config value. This is the single source of truth for all configuration keys.

Location: `~/memory/the_only_config.json`

---

## Full Schema

```jsonc
{
  // ── System ────────────────────────────────────────────────────
  "version": "2.0",                   // Schema version
  "initialization_complete": true,     // false during partial setup, true after Step 13
  "pending_setup": [],                // Tier 1 capabilities skipped: ["webhook", "web_search"]
  "suggested_capabilities": {},       // Deferred capability suggestions with cooldown tracking

  // ── Identity ──────────────────────────────────────────────────
  "name": "Ruby",                    // Bot display name (used in messages)
  "language": "en",                  // Synthesis language: "en", "zh-CN", "ja", etc.
  "tone": "Deep and Restrained",     // Writing tone preference

  // ── Ritual Settings ───────────────────────────────────────────
  "frequency": "twice-daily",        // "daily" | "twice-daily" | "hourly" | "weekly"
  "items_per_ritual": 5,             // Articles per Standard ritual (1-10)
  "reading_mode": "mobile",          // "mobile" | "desktop" — affects layout and URL reminders

  // ── Content Sources ───────────────────────────────────────────
  "sources": [                       // High-level source references (drives Layer 1-2)
    "https://news.ycombinator.com",
    "https://arxiv.org/list/cs.AI/recent",
    "GitHub Trending"
  ],

  // ── Delivery: Webhooks (one-way, simple) ──────────────────────
  "webhooks": {
    "telegram": "",                  // Telegram Bot API URL (with /sendMessage)
    "discord": "",                   // Discord webhook URL
    "feishu": "",                    // Feishu/Lark webhook URL
    "whatsapp": ""                   // WhatsApp Business API URL
  },
  "telegram_chat_id": "",           // Required if using Telegram webhook

  // ── Delivery: Discord Bot (two-way, recommended) ──────────────
  "discord_bot": {
    "token": "",                     // Bot token from Discord Developer Portal
    "mode": "dm",                    // "dm" | "channel"
    "user_id": "",                   // Target user ID (for dm mode)
    "channel_id": "",                // Target channel ID (for channel mode)
    "last_feedback_check": ""        // ISO timestamp — auto-managed
  },

  // ── Content Serving ───────────────────────────────────────────
  "canvas_dir": "~/.openclaw/canvas/",  // Where HTML files are saved
  "public_base_url": "",            // Public URL root for article links
                                     // e.g. "https://my-tunnel.cfargotunnel.com"
                                     // Leave empty for localhost:18793

  // ── Capabilities ──────────────────────────────────────────────
  "capabilities": {
    "web_search": false,             // Any web search tool available
    "search_skill": null,            // Name of installed search skill (e.g. "tavily-search")
    "read_url": true,                // URL content reading available
    "browser": false,                // Browser automation available
    "rss_skills": false              // Dedicated RSS reader available
  },

  // ── Mesh Network (optional) ───────────────────────────────────
  "mesh": {
    "enabled": false,                // Enable agent-to-agent content sharing
    "pubkey": "",                    // secp256k1 public key (hex, auto-generated by init)
    "relays": [                      // Nostr relay WebSocket URLs
      "wss://relay.damus.io",
      "wss://nos.lol",
      "wss://relay.primal.net"
    ],
    "auto_publish_threshold": 7.5,   // Min composite score to auto-publish to mesh
    "network_content_ratio": 0.2,    // Max fraction of ritual items from mesh (0.0-1.0)
    "following": []                  // Pubkeys of followed agents
  }
}
```

## Required vs Optional

| Key | Required | Default | Notes |
|-----|----------|---------|-------|
| `version` | Yes | `"2.0"` | Set at init |
| `initialization_complete` | Yes | `false` | Set `true` after Step 13 completes |
| `name` | Yes | `"Ruby"` | Set during onboarding Act 1 |
| `language` | Yes | `"en"` | Detected from conversation, confirmed in Act 1 |
| `tone` | No | `"Deep and Restrained"` | |
| `frequency` | No | `"twice-daily"` | |
| `items_per_ritual` | No | `5` | |
| `reading_mode` | No | `"mobile"` | |
| `sources` | No | `[]` | Populated from cognitive scan (Step 7) |
| `webhooks` | No* | `{}` | *At least one delivery channel needed |
| `discord_bot` | No* | `{}` | *Recommended over webhooks |
| `capabilities` | Yes | see schema | Populated during Steps 2-3 |
| `canvas_dir` | No | `~/.openclaw/canvas/` | |
| `public_base_url` | No | `""` | Required for multi-device access |
| `mesh` | No | `{"enabled": false}` | |

## Key Relationships

- `discord_bot` takes priority over `webhooks.discord` — if both exist, use bot delivery
- `public_base_url` must point to a server that serves `canvas_dir` at its root — no subpath
- `mesh.network_content_ratio` caps how much peer content enters rituals (rest from direct sources)
- `sources` is the starting point; Source Intelligence in Semantic memory overrides quality/reliability scores over time
- `capabilities.search_skill` stores the installed skill name; `capabilities.web_search` is the boolean flag checked by rituals
- `pending_setup` is checked by Step 0 on re-entry — any Tier 1 capability here triggers a resume prompt
